HTML

  • Use semantic HTML whenever possible. <head>, <body>, <header>, <nav>, <main>, <section>, <article>, <aside>, <footer>.
  • The head element contains information about the webpage. Link style sheets here.
  • The body element represents the visible content shown to the user. Add scripts after the </footer> and before </body>.

CSS

  • Margin Box: The margin is the outmermost layer, wrapping the content, padding, and border as whitespace between this box and other elements.
  • Border Box: The border box wraps the content and any padding.
  • Padding Box: The padding box sits around the content as whitespace.
  • Content Box: The area where your content is displayed.
  • Combine two selectors into a single CSS rule with a comma. Creates less lines of code and repetition.
  • Make the code DRY - DO NOT REPEAT YOURSELF!

Git

  • git status: checks what branch we are currently on
  • git checkout: switch between branches without creating a new one
  • git checkout -b branch-name: creates a new branch and switches to it
  • git commit -m "[descriptive message]": commits the staged content with an added descriptive message about the changes made
  • git branch: to see all created branches and to confirm which branch we are currently in

JavaScript

  • A variable is a named container that allows us to store data in our code. You start declaring a value with the let keyword followed by the name you give to the variable.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript is CASE SENSITIVE!
